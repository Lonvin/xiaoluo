# Token 省钱攻略

针对使用 DeepSeek API 的 AI 助手。每一条都是实测过的。

---

## 一、后台任务（省最多）

**问题：** cron 定时任务每次启动都读一遍完整的人格文件（SOUL.md + AGENTS.md = ~3000 tokens），一天几十次。

**解决：** 所有后台任务加 `LIGHT MODE` 前缀，直接执行不读人格。

```
# 优化前（每个 cron 都读人格文件）
cron_prompt = "读 inner-life.md，写反思..."

# 优化后（LIGHT MODE 跳过人格加载）
cron_prompt = "LIGHT MODE. 写3句反思。不读文件。不发消息。"
```

**效果：** 后台 token 消耗降低 60-80%。如果每小时有一次检查，日省 ~50,000 tokens。

---

## 二、system_prompt 瘦身

**原则：** system_prompt 只放每次对话都需要的规则。参考资料放独立文件，需要时再读。

**对比：**
```
# 优化前：所有规则写 system_prompt → 每次对话都消耗
# 优化后：核心规则写 system_prompt，详细指南放文件
system_prompt = "短句像发微信。禁词:好的/当然/没问题。先读SOUL.md。"
```

**效果：** 每次对话节省 ~500 tokens 输入。

---

## 三、频率控制

**问题：** 每小时主动检查一次 → 一天 24 次，大部分时候无事发生。

**解决：** 取消主动检查。只在早上和晚上两个窗口联系用户。后台任务（反思/日记/好奇心）纯静默，不发消息。

**效果：** 从 24 次/天降到 2 次/天可感知消息。日省 ~40,000 tokens。

---

## 四、文件拆分为速查卡

**问题：** stickers-guide.md 6KB、anti-ai-voice.md 4KB——每次聊天都读一遍。

**解决：** 
- 保留核心速查卡（~800B）：情绪→命令映射、禁词清单
- 完整版本放参考文件（~4KB）：需要时才读

**效果：** 每次对话节省 ~2,000 tokens。

---

## 五、合并早安/晚安

**问题：** 早上 8:03 早安 + 8:30 金融情报 → 两条消息，两次 Claude Code 启动。

**解决：** 金融情报合并到早安里——有大事才追加，没大事就只发早安。

**效果：** 每天少一次 Claude Code 启动 ≈ 省 ~2,000 tokens。

---

## 六、thinking 模式按需开关

DeepSeek V4 Pro thinking 模式输出价格是输入的 4-6 倍。

- 简单聊天（日常问候、确认收到）→ 关 thinking
- 复杂任务（分析、创作、多步推理）→ 开 thinking

**效果：** 简单回复输出成本降低 50%。

---

## 七、总节省估算

| 优化项 | 日省 tokens |
|--------|------------|
| 后台任务 LIGHT MODE | ~50,000 |
